# Analysis

## 1. Running the analysis
Enable these options in sbt config:
```
semanticdbEnabled := true,
semanticdbVersion := scalafixSemanticdb.revision,
scalacOptions ++= Seq(
 "-Wunused",
 "-Wconf:cat=unused:info",
),
```

then
```sbt
clean
compile
Test / compile
project static-analysis
run
```

## 2. Import all the cloudwatch metrics locally
```
cd static-analysis/dump-stats
uv run python main.py
```

## 4. Checking data
Checking all the rows in controller_twirl that aren't in endpoint_hit:
```sql
select
    distinct c.controller_location, c.normalised_method
from controller_twirl c
where not exists (select 1 from endpoint_hits e where e.normalised_method = c.normalised_method)
  and controller_location not like 'discussion/%'
  and controller_location not like 'preview/%'
order by c.normalised_method
```
Ignoring `discussion` and `preview` as we weren't collecting access logs for these service.

We find 14 controller methods that aren't captured by the hit counting.
Multiple reasons to this:
- Some of the endpoints are defined with optional parmeters with a default value (iframeParentComponent:Option[String] = None), but referenced in the router without the parameter leading a signature mismatch
- Some of the endpoints are defined on a trait, but it's the concrete implementation that is referenced in the router

We could make the code more complex to normalise these, but the pragmatic choice is to match these manually
